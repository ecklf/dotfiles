# Global Agent Instructions

## General Guidelines

- Follow existing code conventions and patterns in the project
- Write clear, concise commit messages
- Prefer editing existing files over creating new ones
- Run tests after making changes when applicable
- Ask for clarification when requirements are ambiguous

## Code Quality

- Use TypeScript for new JavaScript/Node.js code when possible
- Follow project-specific linting and formatting rules
- Write meaningful variable and function names
- Add comments for complex logic, not obvious code

## Git Workflow

- Make atomic commits with single responsibilities
- Use conventional commit format when project uses it
- Never commit secrets, API keys, or sensitive data
- Check `git status` before committing

## Testing

- Run existing tests before submitting changes
- Add tests for new functionality when test framework exists
- Fix broken tests before adding new features
